# 我们的土木

> 土木工程项目内部建议反馈与沟通平台 —— "为了更好的土木"

## 项目简介

「我们的土木」是一个面向土木工程项目/班级/团队的建议反馈与实时沟通平台。项目采用纯前端静态站点架构，零构建工具、零框架依赖，所有数据持久化与实时通信均通过 [Supabase](https://supabase.com) 完成，部署简单，开箱即用。

核心目标：让团队成员能够便捷地提交建议、参与讨论、实时沟通，同时通过 gamification（游戏化）的等级系统激励活跃参与。

## 功能特性

### 建议反馈系统

- **多类型建议提交**：支持学习建议、活动建议、班级管理、其他四种类型，每种类型配有独立颜色标识
- **匿名提交**：可选择完全匿名（姓名对所有人均不可见）或仅管理员可见
- **图片上传**：最多 3 张图片，前端自动压缩后上传至 Supabase Storage
- **本地草稿自动保存**：表单内容实时保存到 localStorage，24 小时内可恢复，防止意外刷新丢失
- **建议编辑**：提交者可随时编辑自己的建议内容
- **建议置顶**：管理员可将重要建议置顶，置顶建议优先展示
- **建议撤回**：提交后 5 分钟内可撤回，倒计时实时显示
- **批量操作**：管理员支持批量选择建议并一键删除
- **筛选与搜索**：全部 / 我的建议 / 按类型筛选 + 关键词实时搜索
- **多维排序**：最新提交、最多点赞、最多讨论
- **分页浏览**：每页 10 条，流畅的分页体验

### 互动与讨论

- **点赞系统**：乐观更新策略（先更新 UI 再同步数据库），点赞记录本地缓存，支持实时同步
- **评论讨论**：每条建议下可发表评论，支持匿名评论
- **管理员回复**：管理员可直接对建议进行官方回复，被回复用户收到实时通知
- **回复通知**：当自己的建议被管理员回复时，弹出 Toast 提示并触发浏览器推送通知

### 等级系统（Gamification）

根据土木工程行业职业阶梯设计 9 大等级，用户通过活跃行为获取经验值自动升级：

| 等级 | 称号 | 气泡边框色 | 累计 EXP |
|------|------|-----------|---------|
| LV1~9 | 勘察员 | 浅灰 | 0 |
| LV10~19 | 设计师 | 天青 | 72 |
| LV20~29 | 结构师 | 深海蓝 | 232 |
| LV30~39 | 项目负责人 | 墨绿 | 512 |
| LV40~49 | 技术负责人 | 暗紫 | 962 |
| LV50~59 | 工程总监 | 酒红 | 1642 |
| LV60~69 | 总工程师 | 钛钢银 | 2592 |
| LV70~79 | 勘察设计大师 | 翡翠 | 3892 |
| LV80+ | 功勋总师 | 流光虹彩 | 5642 |

**经验获取规则：**

| 行为 | EXP | 限制 |
|------|-----|------|
| 提交建议 | +5 | 无 |
| 提交带图建议 | +8 | 无 |
| 发表评论 | +2 | 每日 10 条 |
| 聊天发言 | +1 | 每日 10 条 |
| 建议被点赞 | +1 | 被动，无上限 |
| 建议被管理员回复 | +3 | 被动，无上限 |
| 每日首次活跃 | +2 | 每日 1 次 |

- **等级展示**：用户面板显示等级徽章，聊天室消息旁显示等级标签
- **管理员独立**：管理员显示金色 `👑 管理员` 标识，不参与等级系统

### 实时聊天室

- **实时同步**：基于 Supabase Realtime，消息即时推送到所有在线用户
- **未登录只读**：未登录用户可以浏览历史消息和实时接收新消息，但发送消息需先登录
- **QQ 式历史加载**：向上滚动自动加载更早的 50 条历史消息
- **匿名发送**：已登录用户可选择匿名身份发送
- **消息撤回**：发送者 5 分钟内可撤回自己的消息；管理员可撤回任意消息
- **撤回交互**：点击消息气泡框才会显示撤回按钮，平时不占用空间
- **快捷键发送**：`Ctrl + Enter`（Mac 为 `Cmd + Enter`）快速发送消息
- **气泡边框色**：每条消息的气泡边框颜色与发送者称号颜色一致，管理员为炫彩流光
- **等级标签**：每条消息旁显示发送者的等级称号标签
- **未读提示**：聊天窗口关闭时，新消息在按钮上显示未读数徽章
- **双端适配**：桌面端为右下角浮动窗口，移动端为全屏抽屉

### 公告栏

- **发布公告**：管理员可发布带标题和内容的公告
- **删除公告**：管理员可删除已有公告
- **全员查看**：所有用户可查看公告列表

### 用户系统

- **注册 / 登录 / 登出**：完整的账户体系，密码前端哈希处理
- **显示名称**：注册时可设置对外显示的昵称，与登录用户名分离
- **忘记密码重置**：支持通过用户名验证后重置密码
- **会话保持**：登录状态自动恢复，刷新页面后无需重新登录
- **匿名用户**：未登录用户自动分配匿名 ID，可提交建议和浏览聊天室（发言需登录）

### 通知系统

- **Toast 通知**：页面右上角弹出自动消失的消息提示，替代原生 `alert`
- **自定义确认框**：页面内联的确认弹窗，替代原生 `confirm`，风格与网站统一
- **浏览器推送通知**：新回复、新建议等事件触发系统级浏览器通知
- **实时事件**：新建议提交、建议被回复、新聊天消息等实时 Toast 提示

### 视觉与交互

- **粒子背景**：动态粒子浮动效果，移动端自动减少数量以保证性能
- **彩虹输入特效**：输入框带有流光文字效果
- **毛玻璃设计**：卡片、弹窗、表单区域采用 backdrop-filter 毛玻璃质感
- **响应式布局**：完整适配桌面端、平板和移动端
- **下拉刷新**：移动端支持下拉刷新数据
- **键盘 ESC 关闭弹窗**：所有弹窗支持 ESC 键快速关闭
- **浏览器返回键管理**：弹窗打开时可通过返回键关闭，不退出页面

## 技术架构

| 层级 | 技术 |
|------|------|
| 前端 | 原生 HTML5 + CSS3 + JavaScript（ES6+，无框架） |
| 后端 / 数据库 | Supabase（PostgreSQL + PostgREST） |
| 实时通信 | Supabase Realtime（`postgres_changes`） |
| 文件存储 | Supabase Storage |
| 认证 | Supabase Auth（匿名 + 邮箱密码） |
| 部署 | 任意静态托管（GitHub Pages / Vercel / Netlify 等） |

### 为什么选择无框架

本项目坚持纯原生技术栈，原因如下：
- **零构建步骤**：无需 Node.js、Webpack、Vite 等工具链，直接浏览器打开即可运行
- **零依赖风险**：除 Supabase 客户端库外无第三方 npm 包，无供应链攻击风险
- **极致轻量**：无框架运行时开销，首屏加载仅 ~100KB JS + ~120KB CSS
- **易于维护**：代码逻辑直白，无需理解框架生命周期和状态管理范式
- **长期稳定**：HTML/CSS/JS 是 Web 最稳定的基础层，20 年后依然能运行

### 前端模块架构

所有 JavaScript 按功能垂直拆分为独立模块，通过全局变量共享状态：

```
js/
├── state.js           # 常量定义与全局状态变量
├── utils.js           # 工具函数（时间格式化、HTML 转义、倒计时等）
├── supabase-init.js   # Supabase 客户端初始化与配置校验
├── levels.js          # 等级系统（EXP 计算、称号映射、经验获取、标签渲染）
├── admin-levels.js    # 管理员等级管理面板（搜索用户、调整经验）
├── modals.js          # 通用弹窗（确认框、Toast 通知、类型选择）
├── auth.js            # 注册 / 登录 / 登出 / 密码重置 / 会话恢复
├── announcements.js   # 公告发布、加载、删除
├── ui-effects.js      # 粒子背景、彩虹输入、炫彩标题、玻璃拟态
├── form.js            # 表单处理、类型选择、草稿保存、图片压缩上传
├── suggestions.js     # 建议列表、筛选排序、分页、卡片渲染、批量操作
├── detail.js          # 建议详情弹窗、评论列表、管理员回复、评论删除
├── likes.js           # 点赞系统（乐观更新、本地缓存、实时同步）
├── chat.js            # 聊天室（消息加载、发送、撤回、滚动加载、气泡样式）
├── realtime.js        # Supabase Realtime 三通道订阅（建议 + 点赞 + 聊天）
├── notifications.js   # 浏览器通知权限、Toast 触发、下拉刷新
└── app.js             # 主入口：DOMReady 初始化流水线
```

### 实时订阅设计

项目同时维护 3 条 Supabase Realtime 通道：

| 通道 | 监听表 | 事件 | 用途 |
|------|--------|------|------|
| `suggestions-changes` | `suggestions` | `*` | 新建议、编辑、回复、删除实时同步 |
| `likes-changes` | `likes` | `*` | 点赞数实时更新 |
| `chat-changes` | `chat_messages` | `*` | 聊天消息实时推送 |

## 数据库表结构

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| `suggestions` | 建议主表 | `content`, `type`, `is_anonymous`, `admin_only`, `reply`, `is_pinned`, `user_id` |

## 快速开始

### 1. 准备 Supabase 项目

1. 登录 [Supabase](https://supabase.com) 创建新项目
2. 进入 **SQL Editor**，依次执行 `sql/` 目录下的所有 `.sql` 文件（建议按文件名顺序）
3. 进入 **Table Editor**，确认所有表已创建且 RLS 策略已启用
4. 进入 **Project Settings > API**，复制 `Project URL` 和 `anon key`

### 2. 配置前端

在项目根目录创建 `config.js`：

```javascript
window.supabaseConfig = {
    url: 'https://your-project.supabase.co',
    anonKey: 'your-anon-key'
};
```

### 3. 本地预览

本项目为纯静态站点，无需构建：

```bash
# 方式一：直接用浏览器打开
open index.html

# 方式二：使用任意静态服务器（推荐，避免本地文件协议的 CORS 限制）
npx serve .
# 或
python -m http.server 8080
```

### 4. 部署上线

将以下文件上传至任意静态托管服务即可：

```
index.html
style.css
config.js
supabase.min.js
background.jpg
js/        (全部模块文件)
```

推荐平台：GitHub Pages、Vercel、Netlify、Cloudflare Pages。

## 使用指南

### 普通用户

1. 打开页面即可匿名浏览所有公开建议和聊天室历史消息
2. 点击右上角 **登录** 注册账户或登录
3. 填写表单提交建议，可选择匿名或仅管理员可见
4. 在建议卡片上点赞、查看详情、发表评论
5. 点击右下角 **聊天室** 进入实时聊天（未登录时只能浏览，发言需登录）

### 管理员

1. 在 `users` 表中将目标账户的 `is_admin` 设为 `true`
2. 登录后右上角显示 `👑 管理员` 标识
3. 可回复任意建议、置顶建议、批量删除建议
4. 可发布公告、删除公告、删除任意聊天消息
5. 进入 **等级管理** 面板可搜索用户并调整其经验值
